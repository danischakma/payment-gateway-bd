My Gateway কী?

এটি একটি ম্যানুয়াল পেমেন্ট গেটওয়ে যা বিকাশ ও নগদ-এর মাধ্যমে পেমেন্ট গ্রহণ করে। ইউজার Send Money করলে SMS আসে, সেই SMS অটোমেটিক ভেরিফাই হয়ে অর্ডার কমপ্লিট হয়।

তিনটি জিনিস দরকার: ① Render.com-এ ডিপ্লয় ② F-Droid SMS Forwarder অ্যাপ (Android) ③ আপনার ওয়েবসাইটের সাথে লিংক

কাজের ধারা

পেমেন্ট কিভাবে কাজ করে

①
ইউজার পেমেন্ট শুরু করে
আপনার সাইট থেকে গেটওয়ে URL-এ পাঠানো হয়: /?order_id=X&product=Y&amount=Z
②
বিকাশ/নগদে Send Money করে
গেটওয়ে নাম্বার দেখায়। ইউজার মোবাইল অ্যাপ দিয়ে টাকা পাঠায়।
③
TRX ID সাবমিট করে
পেমেন্টের পর SMS আসে। ইউজার সেই TRX ID গেটওয়েতে দেয়।
④
F-Droid SMS Forwarder অটো SMS পাঠায়
আপনার ফোনে আসা বিকাশ/নগদ SMS টি অ্যাপ স্বয়ংক্রিয়ভাবে গেটওয়েতে পাঠায়।
⑤
পেমেন্ট কনফার্ম
গেটওয়ে TRX ID ম্যাচ করে অর্ডার completed করে। আপনার সাইটের callback_url-এ নোটিফিকেশন যায়।

ধাপ ১ — PostgreSQL Database তৈরি করুন

Render-এর ফ্রি PostgreSQL ডেটাবেজ তৈরি করুন। এটি আপনার অর্ডার ও SMS লগ সংরক্ষণ করবে।

প্রথমে Database তৈরি করুন, তারপর Web Service। কারণ Web Service-এ DATABASE_URL লাগবে।

১
Render.com-এ অ্যাকাউন্ট তৈরি করুন
render.com সাইটে যান → Get Started for Free বাটনে ক্লিক করুন → GitHub বা Google দিয়ে সাইনআপ করুন।
২
New PostgreSQL তৈরি করুন
Dashboard-এ যান → New + বাটনে ক্লিক করুন → PostgreSQL সিলেক্ট করুন।
৩

ধাপ ৩ — Environment Variables সেট করুন

Web Service তৈরির পেজে নিচে Environment Variables সেকশনে নিচের ভ্যালুগুলো একে একে Add করুন:

KeyValue (উদাহরণ)বিস্তারিত
DATABASE_URL আবশ্যক postgresql://pguser:xxxx@dpg-xxx.singapore-postgres.render.com/paymentdb ধাপ ১ এ কপি করা External Database URL
ADMIN_TOKEN আবশ্যক MySecret@2024 অ্যাডমিন প্যানেলের পাসওয়ার্ড — যা খুশি দিন, মনে রাখবেন
BASE_URL আবশ্যক https://my-payment-gateway.onrender.com আপনার Render App URL (deploy হওয়ার পরে পাবেন)
BKASH_NUMBER আবশ্যক 01712345678 যেই বিকাশ নম্বরে টাকা আসবে
NAGAD_NUMBER আবশ্যক 01812345678 যেই নগদ নম্বরে টাকা আসবে
ALLOWED_ORIGINS ঐচ্ছিক https://yoursite.com আপনার সাইটের URL (CORS এর জন্য। খালি রাখলে সব সাইট থেকে অ্যাক্সেস হবে)

গুরুত্বপূর্ণ: প্রথমবার BASE_URL পাবেন না — Deploy করুন, URL পাবেন, তারপর BASE_URL আপডেট করুন। এটি সেভাবেও কাজ করবে।

SMS ফরওয়ার্ডিং

কেন SMS Forwarder দরকার?

বিকাশ/নগদে পেমেন্ট হলে আপনার ফোনে একটি SMS আসে। সেই SMS-এ TRX ID থাকে। এই TRX ID গেটওয়েতে পাঠাতে হয় যাতে পেমেন্ট অটোমেটিক কনফার্ম হয়। SMS Forwarder অ্যাপ এই কাজটি স্বয়ংক্রিয়ভাবে করে।

যেই ফোন নম্বরে বিকাশ/নগদ রেজিস্ট্রেশন করা সেই ফোনে SMS Forwarder অ্যাপ ইন্সটল করতে হবে। ফোনটি সবসময় ইন্টারনেটে কানেক্টেড থাকতে হবে।

F-Droid SMS Forwarder — ধাপে ধাপে সেটআপ

ধাপ ১: F-Droid ও অ্যাপ ইন্সটল করুন

১
F-Droid ডাউনলোড করুন
Android ব্রাউজারে যান → f-droid.org → Download F-Droid বাটনে ক্লিক করুন।

APK ডাউনলোড হবে। ইন্সটল করতে "Unknown Sources" থেকে ইন্সটল Allow করতে হতে পারে:
Settings → Security → Unknown Sources → Enable
২
SMS Forwarder খুঁজুন ও ইন্সটল করুন
F-Droid অ্যাপ খুলুন → Search বাটনে ট্যাপ করুন → "SMS Forwarder" লিখুন → bogdandonduk এর অ্যাপটি সিলেক্ট করুন → Install করুন।

বা সরাসরি F-Droid থেকে APK ডাউনলোড করুন:
f-droid.org/en/packages/bogdandonduk.smsforwarder/

ধাপ ২: অ্যাপ কনফিগার করুন

১
Webhook URL প্রস্তুত করুন
আপনার গেটওয়ে URL + /api/webhook/sms — যেমন:

https://my-payment-gateway.onrender.com/api/webhook/sms
এই URL টি কপি করুন — অ্যাপে পেস্ট করতে হবে।
২
SMS Forwarder অ্যাপ খুলুন ও পারমিশন দিন
অ্যাপ খুলুন → SMS পড়ার পারমিশন Allow করুন।

যদি manually দিতে হয়: Android Settings → Apps → SMS Forwarder → Permissions → SMS → Allow
৩
নতুন Forwarder/Rule তৈরি করুন
অ্যাপে + বা Add বাটনে ট্যাপ করুন।

বিকাশের জন্য:
  • Filter/Sender: bKash
  • Forward To: HTTP/Webhook
  • URL: আপনার Webhook URL
  • Method: POST
নগদের জন্য: আরেকটি Rule তৈরি করুন, Sender: Nagad
৪
JSON Body সেট করুন
Body Format: JSON
Request Body: নিচের যেকোনো একটি (অ্যাপ যেটা সাপোর্ট করে):
// বিকল্প ১ (বেশিরভাগ অ্যাপে কাজ করে): { "message": "%message%", "sender": "%sender%" } // বিকল্প ২ (F-Droid SMS Forwarder এর ডিফল্ট): { "text": "%message%", "from": "%sender%" }
গেটওয়ে উভয় ফরম্যাট সাপোর্ট করে। অ্যাপে যেটা দেখায় সেটাই ব্যবহার করুন। %message% ও %sender% হলো অ্যাপের placeholder।
৫
Save করুন ও Enable করুন
Rule save করুন। Toggle/Switch দিয়ে Enable করুন।

ধাপ ৩: Battery Optimization বন্ধ করুন (গুরুত্বপূর্ণ!)

Battery Optimization চালু থাকলে Android Background এ অ্যাপটি বন্ধ করে দেয়। তাহলে SMS ফরওয়ার্ড হবে না!

১
Battery Optimization বন্ধ করুন
Android Settings → Battery → Battery Optimization → SMS Forwarder সার্চ করুন → "Don't optimize" সিলেক্ট করুন।

Samsung হলে: Settings → Apps → SMS Forwarder → Battery → Unrestricted
২
Autostart Allow করুন (যদি থাকে)
কিছু Android ভার্শনে Autostart সেটিং থাকে:
Settings → Autostart বা Startup Manager → SMS Forwarder → Enable
৩
অ্যাপটি Background Service হিসেবে চলার Allow দিন
SMS Forwarder খুলুন → Settings → Run in Background বা Persistent Notification Enable করুন। Notification bar-এ অ্যাপের icon থাকলে বুঝবেন চলছে।

ধাপ ৪: টেস্ট করুন

১
ম্যানুয়ালি টেস্ট SMS পাঠান
Postman বা নিচের curl কমান্ড দিয়ে সরাসরি webhook টেস্ট করুন:
curl -X POST https://YOUR-APP.onrender.com/api/webhook/sms \ -H "Content-Type: application/json" \ -d '{"message": "You have received Tk 500.00 from 01712345678. TRX ID: ABCD1234. Balance Tk 1500.00", "sender": "bKash"}'
২
অ্যাডমিন SMS লগ চেক করুন
/admin পেজে লগইন করুন → SMS লগ ট্যাবে যান। SMS লগে দেখা গেলে কানেকশন সফল।
৩
রিয়েল পেমেন্ট টেস্ট করুন
একটি অর্ডার তৈরি করুন → বিকাশ/নগদ থেকে ১ টাকা Send Money করুন → TRX ID সাবমিট করুন → পেমেন্ট অটো কনফার্ম হয় কিনা দেখুন।

মনে রাখুন: ফোনটি সবসময় চালু ও ইন্টারনেটে সংযুক্ত রাখতে হবে। বিদ্যুৎ গেলে UPS ব্যবহার করুন বা চার্জে রাখুন।

আপনার ওয়েবসাইটের সাথে সংযোগ

মাত্র একটি লিংক বা API কল দিয়ে যেকোনো ওয়েবসাইটের সাথে সংযুক্ত করতে পারবেন।

const res = await fetch('https://YOUR-APP.onrender.com/api/initiate', { method: 'POST', headers: {'Content-Type': 'application/json'}, body: JSON.stringify({amount:500, product_name:'My Product', callback_url:'https://yoursite.com/cb'}) }); const data = await res.json(); if (data.success) window.location(data.payment_url);
import requests r = requests.post('https://YOUR-APP.onrender.com/api/initiate', json={'amount':500,'product_name':'My Product','callback_url':'https://yoursite.com/cb'}) data = r.json() if data['success']: return redirect(data['payment_url'])

FAQ

Render Free টিয়ারে কি ২৪/৭ কাজ করে?
না। Render Free টিয়ারে ১৫ মিনিট inactive থাকলে sleep হয়ে যায়। প্রথম request-এ ৩০-৬০ সেকেন্ড লাগে। ২৪/৭ চালু রাখতে Starter Plan ($7/মাস) নিন। অথবা UptimeRobot দিয়ে প্রতি ১৫ মিনিটে ping করুন।
SMS Forwarder অ্যাপ SMS পাঠাচ্ছে না কেন?
① Battery Optimization বন্ধ করুন ② SMS পড়ার Permission দিন ③ ইন্টারনেট সংযোগ আছে কিনা দেখুন ④ অ্যাপে Rule Enable আছে কিনা দেখুন ⑤ Webhook URL সঠিক কিনা চেক করুন।
পেমেন্ট TRX ID ম্যাচ হচ্ছে না কেন?
① Admin → SMS লগ-এ যান — SMS আসছে কিনা দেখুন ② TRX ID সঠিকভাবে কপি করা হয়েছে কিনা দেখুন ③ বিকাশের TRX ID সাধারণত 8-10 ক্যারেক্টারের। ④ SMS ফরওয়ার্ড হওয়ার আগে TRX ID দেওয়া হলে polling এর মাধ্যমে পরে ম্যাচ হবে।
Database URL কোথায় পাবো?
Render Dashboard → আপনার PostgreSQL → Connections → External Database URL। এটি postgresql:// দিয়ে শুরু হয়।
একাধিক বিকাশ নম্বর ব্যবহার করা যাবে?
বর্তমানে একটি বিকাশ ও একটি নগদ নম্বর সাপোর্ট করে। একাধিক নম্বর যোগ করতে কোড কাস্টমাইজ করতে হবে।
F-Droid না থাকলে কি অন্য কোনো বিকল্প আছে?
হ্যাঁ। Play Store থেকে "SMS Forwarder" বা "MacroDroid" ব্যবহার করতে পারেন। দুটোই HTTP Webhook সাপোর্ট করে। গেটওয়ে উভয় ক্ষেত্রে কাজ করবে।
