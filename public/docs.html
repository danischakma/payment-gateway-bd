📚 ডকুমেন্টেশন

My Gateway কে আপনার ওয়েবসাইটের সাথে কানেক্ট করুন এবং SMS ফরওয়ার্ডিং সেটআপ করুন।

সহজ ইন্টিগ্রেশন অটো SMS ভেরিফিকেশন নিরাপদ
দ্রুত শুরু

৫ মিনিটে সেটআপ করুন

নিচের ধাপগুলো অনুসরণ করলে আপনার ওয়েবসাইট থেকে সরাসরি বিকাশ/নগদ পেমেন্ট গ্রহণ করতে পারবেন।

এই গেটওয়ে Render.com বা যেকোনো Node.js হোস্টিংয়ে ডেপ্লয় করুন। Render-এ ফ্রি টিয়ারে চালানো যায়।

সেটআপ চেকলিস্ট

১
গেটওয়ে ডেপ্লয় করুন
Render.com-এ একটি নতুন Web Service তৈরি করুন। Environment Variables এ DATABASE_URL, ADMIN_TOKEN, BKASH_NUMBER, NAGAD_NUMBER সেট করুন।
২
অ্যাডমিনে লগইন করুন
/admin পেজে গিয়ে পাসওয়ার্ড দিয়ে লগইন করুন। Settings ট্যাবে বিকাশ/নগদ নাম্বার সেট করুন।
৩
SMS ফরওয়ার্ডিং চালু করুন
নিচের SMS সেটআপ গাইড অনুসরণ করুন। বিকাশ/নগদ SMS অটোমেটিক ফরওয়ার্ড হবে।
৪
আপনার ওয়েবসাইটে লিংক করুন
নিচের ইন্টিগ্রেশন গাইড দেখুন। একটি লিংক বা বাটন দিয়েই কাজ হয়।
কাজের ধারা

পেমেন্ট কিভাবে কাজ করে

①
ইউজার পেমেন্ট শুরু করে
আপনার সাইট থেকে URL প্যারামিটার সহ গেটওয়েতে পাঠান: /?order_id=X&product=Y&amount=Z
②
বিকাশ/নগদে Send Money করে
গেটওয়ে ইউজারকে নাম্বার দেখায়। ইউজার মোবাইল অ্যাপ দিয়ে Send Money করে।
③
TRX ID সাবমিট করে
পেমেন্টের পর SMS আসে। ইউজার সেই TRX ID গেটওয়েতে দেয়।
④
SMS ম্যাচিং ও কনফার্মেশন
SMS ফরওয়ার্ডার অ্যাপ সেই SMS টি Webhook-এ পাঠায়। গেটওয়ে TRX ID ম্যাচ করে অর্ডার কমপ্লিট করে। আপনার সাইটে callback_url-এ নোটিফিকেশন যায়।
ইন্টিগ্রেশন

আপনার ওয়েবসাইটের সাথে সংযোগ

মাত্র একটি লিংক বা API কল দিয়ে আপনার যেকোনো ওয়েবসাইটের সাথে সংযুক্ত করতে পারবেন।

সবচেয়ে সহজ পদ্ধতি: শুধু একটি লিংক তৈরি করুন এবং পেমেন্ট বাটনে সেট করুন। কোনো API কলের দরকার নেই।

পদ্ধতি ১ — সরাসরি লিংক (সবচেয়ে সহজ)

একটি লিংক তৈরি করুন এবং পেমেন্ট বাটনে href হিসেবে ব্যবহার করুন:

HTML
PHP
<!-- পেমেন্ট বাটন --> <a href="https://YOUR-APP.onrender.com/?order_id=ORD-001&product=My+Product&amount=500" target="_blank"> পেমেন্ট করুন </a> <!-- অথবা JavaScript দিয়ে --> <script> function pay(orderId, product, amount) { const url = `https://YOUR-APP.onrender.com/?order_id=${orderId}&product=${encodeURIComponent(product)}&amount=${amount}`; window.open(url, '_blank'); } </script>
<?php $gateway_url = 'https://YOUR-APP.onrender.com'; $order_id = 'ORD-' . time(); $product = 'My Product'; $amount = 500; $pay_url = sprintf( '%s/?order_id=%s&product=%s&amount=%s', $gateway_url, urlencode($order_id), urlencode($product), $amount ); ?> <a href="<?= $pay_url ?>" target="_blank">পেমেন্ট করুন</a>

সার্ভার সাইড থেকে অর্ডার তৈরি করুন এবং পেমেন্ট URL পান:

Node.js

পেমেন্ট সফল হলে নোটিফিকেশন পান (Callback)

Callback URL অবশ্যই আপনার সার্ভারের পাবলিক HTTPS URL হতে হবে। Localhost কাজ করবে না।

পেমেন্ট কমপ্লিট হলে গেটওয়ে আপনার callback_url এ POST request পাঠাবে:

Payload

<?php // yoursite.com/payment/callback.php $body = file_get_contents('php://input'); $data = json_decode($body, true); if ($data['status'] === 'completed') { $order_id = $data['order_id']; // ডেটাবেজে অর্ডার আপডেট করুন mark_order_paid($order_id); } http_response_code(200);
// Express route app.post('/payment/callback', (req, res) => { const { order_id, status } = req.body; if (status === 'completed') { // অর্ডার আপডেট করুন db.updateOrder(order_id, { paid: true }); } res.sendStatus(200); });

API এন্ডপয়েন্ট

সকল API এন্ডপয়েন্টের বেস URL হলো আপনার গেটওয়ে URL। JSON ফরম্যাটে রিকোয়েস্ট ও রেসপন্স।

POST /api/initiate নতুন পেমেন্ট অর্ডার তৈরি

নতুন পেমেন্ট অর্ডার তৈরি করে এবং payment_url ফেরত দেয়।

প্যারামিটারটাইপবিস্তারিত
amount আবশ্যকnumberপেমেন্টের পরিমাণ (টাকা)
product_name ঐচ্ছিকstringপ্রোডাক্টের নাম
callback_url ঐচ্ছিকstringপেমেন্ট সফল হলে POST করার URL
payment_method ঐচ্ছিকstringbkash বা nagad
POST /api/webhook/sms SMS ফরওয়ার্ড রিসিভ

SMS Forwarder অ্যাপ থেকে SMS রিসিভ করে TRX ID ম্যাচ করে।

প্যারামিটারবিস্তারিত
message আবশ্যকSMS এর টেক্সট বডি

SMS ফরওয়ার্ডিং

মেসেজ ফরওয়ার্ডিং সেটআপ

বিকাশ/নগদ পেমেন্টের পর যে SMS আসে, সেই SMS টি স্বয়ংক্রিয়ভাবে গেটওয়েতে পাঠানোর জন্য একটি Android অ্যাপ ব্যবহার করতে হবে।

এটি কেন দরকার? — বিকাশ/নগদ পেমেন্ট হলে আপনার ফোনে একটি SMS আসে। সেই SMS-এ TRX ID থাকে। এই অ্যাপ সেই SMS টি আপনার গেটওয়ে সার্ভারে পাঠিয়ে দেয়, তাহলে পেমেন্ট অটোমেটিক কনফার্ম হয়।

কোন অ্যাপ ব্যবহার করবেন?

নিচের যেকোনো একটি Android অ্যাপ ব্যবহার করতে পারবেন। সবই ফ্রি।

ধাপে ধাপে SMS Forwarder সেটআপ

১
অ্যাপ ডাউনলোড করুন
Google Play Store থেকে "SMS Forwarder" সার্চ করে ইন্সটল করুন। সেই ফোনে ইন্সটল করুন যেই নম্বরে বিকাশ/নগদ আছে।
২
Webhook URL কপি করুন
অ্যাডমিন প্যানেলের Settings ট্যাবে গিয়ে Webhook URL কপি করুন:

https://YOUR-APP.onrender.com/api/webhook/sms
৩
অ্যাপে নতুন Rule তৈরি করুন
অ্যাপ খুলুন → Add Rule বা + বাটনে ক্লিক করুন।
৪
Sender Filter সেট করুন
Sender/From: বিকাশের জন্য bKash লিখুন
নগদের জন্য আলাদা একটি Rule তৈরি করুন, Sender: Nagad

এতে শুধু বিকাশ/নগদের SMS ফরওয়ার্ড হবে, অন্য SMS না।
৫
HTTP Forward কনফিগার করুন
Forward Type: HTTP/Webhook
URL: (উপরের Webhook URL পেস্ট করুন)
Method: POST
Body Format: JSON
Body: {"message": "%message%", "sender": "%sender%"}
৬
পারমিশন দিন
অ্যাপকে SMS পড়ার পারমিশন দিন। Android Settings → Apps → SMS Forwarder → Permissions → SMS: Allow করুন।
৭
টেস্ট করুন
অ্যাডমিন প্যানেলের SMS লগ ট্যাবে গিয়ে একটি টেস্ট SMS পাঠান। লগে দেখা গেলে সফলভাবে কানেক্ট হয়েছে।

গুরুত্বপূর্ণ: ফোনটি সবসময় ইন্টারনেট সংযুক্ত রাখতে হবে। Battery Optimization থেকে SMS Forwarder অ্যাপকে বাদ দিন, নাহলে Background এ বন্ধ হয়ে যেতে পারে।

সাধারণ জিজ্ঞাসা

SMS ফরওয়ার্ডার কোন ফোনে ইন্সটল করবো?
যেই ফোনে আপনার বিকাশ/নগদ রেজিস্টার্ড নম্বর আছে সেই ফোনে। কারণ পেমেন্টের SMS সেই নম্বরেই আসবে।
পেমেন্ট কনফার্ম হতে কতক্ষণ লাগে?
সাধারণত ১০-৩০ সেকেন্ড। SMS আসা → ফরওয়ার্ড হওয়া → TRX ম্যাচ হওয়া — এই পুরো প্রক্রিয়া সাধারণত ৩০ সেকেন্ডের মধ্যে শেষ হয়।
ডেটাবেজ ছাড়া কি চালানো যাবে?
না। সব অর্ডার ও SMS লগ PostgreSQL-এ সংরক্ষিত হয়। Render.com-এ ফ্রি PostgreSQL পাওয়া যায়।
একাধিক বিকাশ/নগদ নম্বর ব্যবহার করা যাবে?
বর্তমানে প্রতিটিতে একটি করে নম্বর সাপোর্ট করে। একাধিক নম্বরের জন্য সোর্স কোড কাস্টমাইজ করতে হবে।
Render.com-এ ডেপ্লয় করার পর অ্যাডমিন পাসওয়ার্ড কোথায় সেট করবো?
Render Dashboard → আপনার Service → Environment → ADMIN_TOKEN ভ্যারিয়েবলে পাসওয়ার্ড সেট করুন।
